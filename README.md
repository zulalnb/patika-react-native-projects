# **Patika React Native Projects**

This repository contains React Native assignments and tutorial projects completed as part of the Patika React Native learning path.

## **About**

The repository is organized as a collection of separate React Native projects. Each folder represents either a course assignment, a practice app, or an implementation from tutorial videos.

## **Projects**

-   **BananeApp** — React Native project with Firebase, navigation, form handling, modals, and date utilities.
-   **codetalks** — Expo Router based React Native project with Firebase, authentication flow, form validation, and modern Expo tooling.
-   **UnitTestIntroApp** — Introductory React Native project focused on unit testing basics with Jest and React Native Testing Library.
-   **ProjectReport** — Project-related report/documentation folder.

## **Tech Stack**

Different projects may use different tools depending on the assignment requirements.

Common technologies used in this repository include:

-   React Native
-   Expo
-   TypeScript
-   React Navigation
-   Expo Router
-   Firebase
-   Jest
-   React Native Testing Library
-   React Hook Form
-   Zod
-   date-fns

## **Getting Started**

Clone the repository:

```bash
git clone https://github.com/zulalnb/patika-react-native-projects.git
```

Navigate into the project you want to run:

```bash
cd patika-react-native-projects/project-folder-name
```

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm start
```

For Expo projects:

```bash
npx expo start
```

For React Native CLI projects:

```bash
npm run android
```

or

```bash
npm run ios
```

## **Purpose**

The main purpose of this repository is to keep React Native course work, assignments, and practice projects in one place while following the Patika.dev learning path.

It also serves as a personal archive for tracking progress in React Native, Expo, Firebase, navigation, forms, validation, and testing.

## **Notes**

Each project may have its own setup requirements. Before running a project, check its `package.json` file and make sure the required environment is configured correctly.
